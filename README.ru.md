# channeljs

Крохотный типобезопасный канал pub/sub: один `Channel<M>` задаётся картой сообщений (MessageMap) и открыт с двух сторон — `tx` для отправки и `rx` для подписки.

| | |
|---|---|
| Версия | 1.1.0 |
| Формат | только ESM (`"type": "module"`) |
| Зависимости | нет |
| Лицензия | MIT |

## Установка

```bash
npm install @evgkch/channeljs
```

## Быстрый старт

```typescript
import Channel from '@evgkch/channeljs';

// Карта сообщений: ключ — событие, значение — кортеж аргументов слушателя.
type Events = {
    ':click': [x: number, y: number];
    ':close': [];
};

const channel = new Channel<Events>();

// `on` возвращает функцию отписки — держать сам слушатель необязательно.
const off = channel.rx.on(':click', (x: number, y: number) => {
    console.log(`клик по ${x}, ${y}`);
});

// Отправка — слушатель выше получит (10, 20).
channel.tx.send(':click', 10, 20);

off();             // отписаться
channel.clear();   // либо снять всех подписчиков сразу
```

## Понятия

| Термин | Что это |
|---|---|
| **MessageMap** | Объект: ключ — сообщение (`string \| number \| symbol`), значение — кортеж аргументов для его слушателей. Например `{ ':click': [x: number, y: number] }`. |
| **Channel`<M>`** | Сам канал pub/sub, параметризованный картой `M`. Хранит подписчиков по каждому сообщению. |
| **`tx`** | Сторона отправки (`Tx<M>`): `send`, `has`. |
| **`rx`** | Сторона приёма (`Rx<M>`): `on`, `once`, `off`. |
| **Off** | Функция отписки `() => boolean`; возвращает `true`, если слушатель был снят. |

## API

### `channel.tx` — отправка

| Метод | Сигнатура | Возвращает |
|---|---|---|
| `send` | `send(msg, ...args)` | `true`, если у сообщения были слушатели |
| `has` | `has(msg)` | `true`, если у сообщения сейчас есть подписчики |

`has` — дешёвая проверка перед отправкой: не собирать тяжёлый payload, если его некому слушать.

### `channel.rx` — подписка

| Метод | Сигнатура | Возвращает |
|---|---|---|
| `on` | `on(msg, listener)` | функцию отписки `Off` |
| `once` | `once(msg, listener)` | функцию отписки `Off` (снимается после первой доставки) |
| `off` | `off(msg, listener)` | `true`, если такой слушатель был |

### `channel` — жизненный цикл

| Член | Описание |
|---|---|
| `tx` / `rx` | две стороны канала |
| `clear()` | снять всех подписчиков (страховка от утечек) |

## Семантика доставки

- **Состав получателей фиксируется на время отправки.** Если во время доставки какой-нибудь обработчик подпишет или отпишет слушателя, текущий `send` этого уже не учтёт — изменение сработает со следующего `send`.
- **Без повторов.** Один и тот же слушатель, подписанный дважды по одной и той же ссылке, регистрируется один раз.
